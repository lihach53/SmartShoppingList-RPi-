# Умный список покупок

<details>
<summary><strong>О проекте</strong></summary>
   
---

Умный список покупок - это Android-приложение с клиент-серверной архитектурой, позволяющее синхронизировать списки покупок между разными устройствами в реальном времени.

---

## Основной функционал

### Android приложение (клиентская часть)
- **Просмотр списка продуктов** - отображение всех товаров для покупки
- **Добавление новых продуктов** - интуитивный интерфейс для добавления позиций
- **Отметка купленных продуктов** - возможность отмечать приобретенные товары
- **Удаление продуктов** - удаление ненужных позиций из списка
- **Автоматическая синхронизация** - своевременное обновление данных между устройствами

###  Raspberry Pi / Raspbian (серверная часть)
- **Хранение данных** - централизованное хранение списка продуктов
- **REST API** - обработка запросов от всех подключенных устройств
- **Real-time синхронизация** - обеспечение актуальности данных на всех клиентах
- **Управление подключениями** - обработка множественных одновременных подключений

---

## Интерфейс приложения

### Основной экран:
1. **Главный экран списка покупок**
   - Отображение всех товаров
   - Статус покупки (куплено/не куплено)
   - Кнопки действий

   - Добавление заметки к товару (опционально)

## Технология

Архитектура системы построена по классической клиент-серверной модели. Серверная часть развернута на Raspberry Pi в VirtualBox (Raspbian) и написана на Python с использованием микрофреймворка Flask. Она предоставляет REST API для выполнения всех операций с данными и использует SQLite в качестве системы хранения. Сервер автоматически инициализирует базу данных при запуске. Были реализованы пять основных endpoint'ов: получение списка товаров, создание нового товара, обновление существующего, удаление товара и проверка статуса сервера. Каждый endpoint включает валидацию входных данных и обработку возможных ошибок, что обеспечивает стабильную работу системы.

Клиентская часть представляет собой нативное Android-приложение, разработанное на Java с использованием современных библиотек и подходов. Для сетевого взаимодействия с сервером применяется Retrofit 2, который значительно упрощает выполнение HTTP-запросов и обработку JSON-ответов. Локальное хранение данных организовано через Room Database, что позволяет в будущем реализовать полноценный офлайн-режим работы приложения. Пользовательский интерфейс соответствует принципам Material Design и включает все необходимые элементы для комфортной работы: список товаров в виде карточек с чекбоксами для отметки покупок, поля для ввода новых позиций и диалоговые окна для редактирования заметок.

Функциональность приложения охватывает все основные потребности пользователя при работе со списком покупок. Пользователь может просматривать полный список товаров, добавлять новые позиции с указанием названия и дополнительных заметок, отмечать купленные товары, редактировать информацию о существующих позициях и удалять ненужные элементы. Особенностью системы стала автоматическая синхронизация данных: любые изменения, внесенные на одном устройстве, немедленно отображаются на всех других устройствах, подключенных к тому же серверу. Дополнительно были реализованы такие улучшения, как отображение даты создания и последнего обновления каждого товара, визуальное выделение купленных позиций через полупрозрачность и форматирование временных меток в удобочитаемый вид.
